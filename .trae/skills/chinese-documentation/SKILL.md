# 中文技术文档写作规范

## 核心原则

排版服务于阅读体验，规范服务于一致性，内容服务于读者。

## 中文排版规范

### 空格
- 中英文之间加空格：`使用 Git 进行版本管理`
- 中文与数字之间加空格：`包含 3 个新功能`
- 数字与单位之间加空格：`不超过 5 MB`
- 例外：度数、百分比不加空格：`32°C`、`95%`

### 标点符号
- 中文语境使用全角标点
- 括号内有英文或数字时用半角括号
- 中文使用直角引号「」

## 术语处理

### 保留英文
- 专有名词：React、Kubernetes、Redis
- 行业通用缩写：API、SDK、CLI、ORM
- 命令和代码：`npm install`
- 没有公认中文翻译的术语

### 翻译为中文
- 有公认翻译的通用概念
- 描述性短语

### 首次出现标注翻译
技术术语首次出现时标注中英对照

## 常见问题

- **机翻味**：避免被动语态、冗余代词、直译英文句式
- **句式欧化**：长句拆成短句，一句话只说一件事
- **过度翻译**：保留业界通用英文术语
- **中英标点混用**：中文句子用全角标点
