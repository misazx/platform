# 工作流执行器

直接在当前会话中执行多角色编排工作流，无需配置 API key。当前 LLM 就是执行引擎。

## 执行流程（5 步）

### 第 1 步：解析工作流
读取 YAML 文件，提取 name、agents_dir、inputs、steps。

### 第 2 步：收集输入
- 必填输入：向用户询问
- 可选输入：使用默认值

### 第 3 步：构建执行顺序
根据 `depends_on` 进行拓扑排序，展示执行计划。

### 第 4 步：逐层执行
- 预读角色文件
- 渲染 task 模板
- 执行（单步骤直接执行，多步骤并行启动子代理）
- 保存输出到上下文

### 第 5 步：保存结果
保存到 `.ao-output/{工作流名称}-{日期}/`

## 重要规则

- 每个步骤都必须真正扮演对应角色
- 角色切换必须明确
- 不要跳过步骤或合并步骤
- 必须先读取角色文件再执行步骤
